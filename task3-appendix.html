Task 3

Heuristics standards

1. Match between system and the real world (language/ icon used in the system is readable for user)

2. Consistency and standard: Keep using one word/object/operation to represent the same thing.

3. Visibility of system status: User could see what is going on when they make every operation. User should be provided with appropriate feedback within a reasonable time.

4. Does the onboarding process make sense to user. Tutorial introduces all the main functions and user could have a basic understanding on system. No confusing or misleading language used.

5. User would always know where they are located within the overall system Does the app allow users to redo & undo (back button).(e.g. go back to the home page whenever they are)

6. Error prevention & recovery from error: Good error message buttons & layout not causing confusion for users If an error occurs, either the app auto-fix it or it instruct users to counter it

7. Aesthetic and intuitive design No irrelevant or rarely needed content for the task being performed.

8. Recognition rather than recall (minimise demands on human memory) Visibility of what each button/page stand for in the system. Information and function should be available at the point where users are likely to need them.

9. Flexibility and efficiency of use All users include people who is not a professional trainer would comprehend. Does it support people who are using it the first time. Accelerator keys available for experienced user

10. Help and documentation design the need for separate help systems
